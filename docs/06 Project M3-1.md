# Технічне завдання: Інтернет-магазин на Django/DRF

## 1. Вступ і мета

Ви — розробник-початківець на своїй першій роботі. Ваше завдання — трансформувати готовий статичний HTML/CSS дизайн на повноцінний застосунок інтернет-магазину, використовуючи Django.

**Проєкт має включати:**

*   **REST API** для зовнішніх клієнтів: DRF, JWT.

**Мета:** продемонструвати розуміння Django, DRF, роботи з базами даних, архітектурних практик, стандартів якості коду та навичок деплою.

## 2. Загальні принципи

*   **Веб-інтерфейс** (для користувачів у браузері) використовує автентифікацію на основі **сесій (session-based)**, що є стандартом для Django.
*   **Зовнішні API** використовують **JWT** для авторизації.
*   **GraphQL** (якщо буде реалізовано) доступний через єдиний ендпоінт `/graphql/`.
*   Структура проєкту є орієнтовною — допускаються варіації, але вона має бути логічною та модульною.
*   **Інфраструктура:** обов'язкове використання **PostgreSQL**. Усі сервіси (застосунок, база даних) мають розгортатися через **Docker Compose**.
*   Використання альтернативних менеджерів залежностей (`poetry`, `pip-tools/uv`) вітається як бонус.

## 3. Функціональні блоки

### 3.1. Каталог товарів та пошук (`/` та `/products/`)

**Функціональність:**

*   Список товарів з пагінацією.
*   Фільтрація за категорією, діапазоном цін.
*   Пошук за назвою та описом.
*   Сортування (за ціною, популярністю, новизною).

**Ключові теми:** CBV, шаблони, ORM, оптимізація запитів до БД.

### 3.2. Сторінка товару (`/product/<slug>/`)

*   Детальна інформація (назва, опис, ціна, зображення, рейтинг).
*   Відгуки з рейтингами (1–5). Можливість залишити відгук (лише після покупки товару — потрібна перевірка).
*   Кнопка "Додати в кошик", вибір кількості.

**Ключові теми:** CBV, форми, ORM, валідація.

### 3.3. Кошик (`/cart/`)

*   Перегляд вмісту кошика.
*   Додавання, видалення, зміна кількості товарів.
*   Підрахунок підсумкової вартості, врахування залишків на складі.
*   Зберігання даних кошика в сесії.
*   Перевірка наявності товару на складі.

**Ключові теми:** Django Sessions, Message Framework, бізнес-логіка.

### 3.4. Оформлення замовлення (`/checkout/`)

*   Форма для контактних даних та адреси доставки.
*   Вибір способу оплати (імітація/мок).
*   Створення запису про замовлення в базі даних.
*   Email-сповіщення для користувача та адміністратора.
*   Обробка помилок та валідація форм.

**Ключові теми:** Форми, надсилання email, транзакції в БД.

### 3.5. Особистий кабінет (`/account/`)

*   Реєстрація, вхід/вихід (session auth).
*   Історія замовлень (з можливістю фільтрації).
*   Редагування профілю.
*   Зміна пароля.
*   (Опційно) Керування адресами доставки.

**Ключові теми:** Автентифікація, ORM, форми, керування доступом.

### 3.6. Адмін-панель (`/admin/`)

*   Керування товарами, категоріями, замовленнями, відгуками, користувачами.
*   Аналітика: агрегації та анотації (виторг, топ-продукти, кількість замовлень).
*   Пошук, фільтри, кастомні дії (actions).
*   Налаштування прав доступу для різних ролей.

**Ключові теми:** Django Admin, агрегації, кастомізація.

### 3.7. REST API (`/api/`)

Приклади ресурсів та дій:

Ресурс        | Дія                    | URL-шаблон                    | HTTP метод(и)            | Опис
--------------|------------------------|-------------------------------|--------------------------|----------------------------------------
**Товари**
              | Список                 | /api/products/                | GET                      | Пагінація, фільтрація, пошук
              | Деталі                 | /api/products/<id>/           | GET                      | Інформація про товар
**Замовлення**
              | Створення              | /api/orders/                  | POST                     | Створення замовлення (на основі кошика)
              | Список своїх           | /api/orders/                  | GET                      | Список замовлень поточного користувача
              | Деталі                 | /api/orders/<id>/             | GET                      | Детальна інформація (лише про своє)
              | Оновлення/скасування   | /api/orders/<id>/             | PATCH/PUT/DELETE         | Зміна статусу/скасування
**Користувачі**
              | Реєстрація             | /api/users/register/          | POST                     | Створити акаунт
              | Вхід (JWT)             | /api/users/login/             | POST                     | Отримати access/refresh токены
**Кошик**
              | Керування              | /api/cart/                    | GET, POST, PATCH, DELETE | Керування вмістом кошика
**Відгуки**
              | Додати/переглянути     | /api/products/<id>/reviews/   | GET, POST                | Список та надсилання відгуку

*   **JWT:** реалізувати механізм access + refresh токенів.
*   **Права доступу:** користувач може переглядати/змінювати лише свої дані.

**Ключові теми:** DRF, серіалізація, ViewSets/routers, пагінація, permissions.

### 3.8. Документація API

*   Swagger/OpenAPI для REST API (наприклад, за адресою `/api/docs/`).
*   Приклади запитів, схеми даних, опис процесу авторизації.

**Ключові теми:** Автоматична генерація документації (drf-spectacular, drf-yasg).

### 3.9. (Бонус) GraphQL

*   Єдиний ендпоінт `/graphql/`.
*   Аналітичні запити:
    *   **Замовлення:** виторг, кількість, середній чек, тренди.
    *   **Продукти:** популярні товари, залишки на складі.
    *   **Користувачі:** активність, кількість повторних покупок.

**Ключові теми:** Graphene-Django, резолвери, авторизація в GraphQL.

> **Примітка:** GraphQL — це бонусне завдання. Його можна замінити іншими покращеннями (наприклад, розширене покриття тестами, налаштування CI/CD тощо).

## 4. Структура даних (База даних)

### Category

*   `id` (PK)
*   `name` (CharField)
*   `slug` (SlugField, unique)
*   `parent` (ForeignKey self, nullable) — для реалізації вкладених категорій
*   `created_at`, `updated_at`

### Product

*   `id` (PK)
*   `name` (CharField)
*   `slug` (SlugField, unique)
*   `description` (TextField)
*   `price` (DecimalField)
*   `category` (ForeignKey to Category)
*   `image` (ImageField)
*   `is_active` (BooleanField)
*   `stock` (IntegerField) — кількість на складі
*   `created_at`, `updated_at`

### Order

*   `id` (PK)
*   `user` (ForeignKey to User)
*   `status` (CharField: `pending`, `paid`, `shipped`, `delivered`, `cancelled`)
*   `total_price` (DecimalField)
*   `shipping_address` (TextField / або окрема модель Address)
*   `created_at`, `updated_at`

### OrderItem

*   `id` (PK)
*   `order` (ForeignKey to Order)
*   `product` (ForeignKey to Product)
*   `quantity` (IntegerField)
*   `price` (DecimalField) — знімок ціни на момент покупки

### Review

*   `id` (PK)
*   `product` (ForeignKey to Product)
*   `user` (ForeignKey to User)
*   `rating` (IntegerField, 1–5)
*   `comment` (TextField)
*   `created_at`

*Додатково можна реалізувати моделі:* Address, Payment тощо.

## 5. Орієнтовна архітектура проєкту

```
/myshop/
├── config/         # Налаштування, urls, asgi/wsgi
├── products/       # Моделі, views, serializers, forms, admin для продуктів
├── orders/         # Логіка замовлень та кошика
├── users/          # Реєстрація, профілі, автентифікація
├── reviews/        # Відгуки (або як частина `products`)
├── payments/       # Способи оплати, вебхуки
├── graphql/        # Схема та резолвери (якщо використовується)
├── templates/      # Django-шаблони
├── static/         # Статичні файли (CSS, JS, images)
├── tests/          # Тести
├── docker-compose.yml # Файл для запуску інфраструктури
├── Dockerfile      # Docker-образ застосунку
├── manage.py
└── README.md
```

> **Примітка:** Структура є орієнтовною. Пріоритет — розподіл відповідальності (Separation of Concerns), читабельність та розширюваність коду.
